Since that's not unlikely to fail, leaving me with an unsusable system, I quickly made a backup of the entire disk using Disk Utility, so that I would be able to restore the entire Mac's disk if something went wrong.

Of course, the installation did not work (it would be "Creating MacPostFactor Recovery (with packages)", only to then reboot, without end).

When I then tried to restore the disk from my backup, Disk Utility would require that I perform the "Scan Image for Restore…" command first.

Imagine my surprise when that failed as well, with the following error message:
Unable to scan "backup.dmg" (Internal error)

I could not even mount the disk image, as that would result in this error:
Unable to attach "backup.dmg" (no mountable file systems)

After some poking around I found that the disk image is intact, however. Why it would neither checksum nor mount is a mystery to me. Maybe it was the fact that I used the compressed option when creating it, though that should still not be a problem.

How would I get the compressed disk image back onto my Mac's disk?

With a non-compressed "plain" disk image, one could simply use the "dd" Terminal command to copy the dmg file back to the disk. But with most dmg files, this won't work. The trick is to let Disk Utility open the disk image without mounting it (mounting did fail, as I wrote above). After that, the "dd" command can be used to write the data back.
